三、端口 #

端口代表计算机上一个程序的进程

端口概念及常识 #

  • 不同的进程有不同的端口号，用来区分软件
  • 端口号范围被规定 0~65535
  • 不同协议相同端口不冲突；单个协议下端口号不能冲突

端口分类 #

  • 共有端口 0~1023

    • HTTP : 80
    • HTTPS : 443
    • FTP : 21
    • SSH : 22
    • Telent : 23

  • 程序注册端口

    • Tomcat : 8080
    • MySql : 3306
    • Oracle : 1521
    • redis : 6379

  • 动态、私有端口

    bash
    netstat -ano  #查看所有的端口
netstat -ano|findstr "5900"  #查看指定的端口
tasklist|findstr "1896"  #查看指定端口的进程

补充：跨域问题 #

  • 协议，端口，主机三者相同即为同源，反之，其中只要某一个不一样则为不同源
  • 同源产生的问题
    • Cookie、LocalStorage 和 IndexDB 无法获取
    • DOM 无法获得
    • AJAX 请求不能发送
  • 本地开发解决跨域方式
    • vite 的 proxy 进行代理
    • 后台开启 cors
    • 使用 nginx 转发请求
  • 生产环境解决跨域方式
    • 后台开启 cors
    • 使用 nginx 转发请求

注意：后台开启cors不需要前端做任何改动、nginx更改配置

四、通信协议 #

网络通信协议：速率、传输码率、代码结构 和 传输控制等

TCP/IP协议簇

应用层：操作系统中具体的程序应用

传输层：网络通信（TCP/UDP）

网络层：数据封包过程

数据链路层：光纤、电缆或卫星传输

TCP（电话） #

  • 连接 稳定
  • 三次握手 四次挥手
  • 客户端 和 服务端
    • 传输完成，连接释放，效率低

UDP（短信） #

  • 不连接 不稳定
  • 客户端、服务端：没有明确的界限
    • 只负责发送，不负责接收与否
